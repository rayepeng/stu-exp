    理解JavaScript的作用域¶

    声明提升¶

    这段代码很经典了

    1
2
3
    a = 2;
var a;
console.log(a);

    很多人觉得输出应该是undefined但是实际上是2

    那看这个：

    1

    同样是输出undefined而不是报错

    先有鸡还是先有蛋¶

    结论： 先有蛋(声明)后有鸡(赋值)

    所有的声明都会提升到最前面执行，也就是那个var a之后再赋值

    更多的栗子¶

    函数中如何表现

    1
2
3
4
5
    foo();
function foo() {
    console.log(a); //undefined
    var a = 2;
}

    这段代码的执行顺序：

    1

    然而函数表达式却不会提升

    1
2
3
4
    foo(); // 此处会导致TypeError 因为foo只是被声明了但是并不知道是一个函数
var foo = function () {
    console.log(1);
}

    1
